# Выводы по работе

## 1. Что было изучено
Я освоил ключевые компоненты `rich`: `Table`, `Progress`, `Panel`, `Syntax`, `Console`. Научился строить структурированный, цветной вывод без ручной работы с ANSI-кодами.

## 2. С какими трудностями столкнулся
Сначала не понимал, зачем нужен объект `Console` — казалось, можно вызывать всё глобально. Позже оценил его гибкость (например, перенаправление вывода в файл). Также потребовалось подобрать подходящие темы и стили.

## 3. Что получилось удачно
Отчёт выглядит профессионально даже при первом запуске. Прогресс-бар анимируется плавно, таблица адаптируется под ширину терминала, код подсвечивается корректно.

## 4. Где инструмент уместен
- CLI-утилиты для разработчиков и админов
- Скрипты генерации отчётов
- Образовательные демонстрации
- Отладка сложных структур (`rich.inspect()`)

## 5. Где инструмент неуместен
- В логах CI/CD без поддержки ANSI
- При экспорте в простой текстовый файл (цвета превратятся в мусор)
- В веб- или GUI-приложениях

## 6. Итоговая оценка
Оценка: **10/10**  
`rich` кардинально меняет восприятие терминального вывода. Простой, мощный, эстетичный. Буду использовать в каждом новом CLI-проекте.
